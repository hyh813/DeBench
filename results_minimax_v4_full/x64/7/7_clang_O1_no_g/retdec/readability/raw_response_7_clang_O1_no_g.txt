# Decompilation Readability Assessment

## Target Function: `param_fake_branch`

### Analysis Overview

The original source demonstrates **OBF-L4-02: Fake Branches (永假)** - always-false conditionals used for code obfuscation. The function contains two never-executed code blocks:
1. `if (x * x < 0)` - mathematically impossible for real numbers
2. `if (strlen(str) < 0)` - string length cannot be negative

The decompiled version **completely eliminates these fake branches**, returning only `a1 & 0xffffffff`. This is a critical semantic loss - the decompiler incorrectly removed what appears to be intentional obfuscation code, likely misinterpreting it as dead code.
