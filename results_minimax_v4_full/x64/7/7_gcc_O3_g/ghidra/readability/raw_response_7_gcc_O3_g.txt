### Top 3 Recommendations for Decompiler Improvement

1. **Infer variable names from usage patterns**: Analyze how variables are used (arithmetic, indexing, comparison) to generate semantically meaningful names. For example, a variable used in `x % y` in a loop is likely a "remainder" or "temp" for gcd calculation.

2. **Detect and annotate optimization artifacts**: When loop unrolling is detected, add a comment like "/* Loop unrolled by compiler */" instead of showing 8 nested if-statitions. This dramatically improves readability at -O3.

3. **Preserve dead code from obfuscation**: The decompiler should flag when code appears to have been optimized away. The fake branches in `param_fake_branch` were intentional obfuscation - the decompiler should note "Dead code eliminated: always-false conditions" rather than silently removing them.
