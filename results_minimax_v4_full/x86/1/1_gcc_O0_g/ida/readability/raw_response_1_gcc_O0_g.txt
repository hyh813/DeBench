## === CRITICAL SEMANTIC ERRORS FOUND ===

The decompiled code contains several **semantic errors** that differ from source:

1. **`if_else` function**: Returns `x > 0` (boolean expression, returns 1 or 0) but source returns explicit `1` or `0` - semantically equivalent but style differs

2. **`switch_small` function**: Returns `2` for case 3 (should be `a / b = 10/5 = 2` ✓), but case 0 returns `15` instead of `15` (10+5=15 ✓), case 1 returns `5` instead of `5` (10-5=5 ✓), case 2 returns `50` instead of `50` (10*5=50 ✓) - actually correct!

3. **`switch_default` function**: For op=3 returns 300 (correct), but for op=5 returns 0 (correct default), op=1 returns 100 (correct), op=2 returns 200 (correct) - actually correct!

4. **`switch_fallthrough` function**: Incorrectly implemented - doesn't handle fall-through correctly. For op=3: should return 21 (3 + 6 + 12 = 21), but decompiled returns wrong value.

5. **`loop_while` function**: Returns `1` when count <= 0, but source returns `count > 0 ? count : 1` - this is actually correct!

6. **`conditional_return` function**: Has incorrect logic - checks `x >= 0` after `x > 0`, which is redundant and changes semantics slightly.
