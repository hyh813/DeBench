### Top 3 Recommendations for Decompiler Improvement
1. **Infer variable names from usage patterns**: When a variable is used in a comparison like `if (size > 1024)`, infer the name `size` or `threshold`. When a variable holds a return value, name it `result` or `return_value`.

2. **Recover macro semantics through constant analysis**: Detect when multiple literals relate (e.g., 1024 and 512 are related by division by 2) and suggest they came from a constant. Recognize patterns like 64 = 16*4 and suggest BUFFER_COUNT * 4.

3. **Add semantic annotations for assembly and atomic operations**: When decompiling inline assembly or atomic operations, add comments explaining what the operation does (e.g., "// Atomic fetch-and-add: returns old value while atomically adding"). Document the calling convention implications.
