# Decompilation Readability Assessment

## Target Function Analysis

**Focus Function:** `param_strcpy` (and related test functions)

The original source shows `param_strcpy` calling `strcpy(dst, src)` and returning `strlen(dst)`. The decompiled version shows:
```c
int __cdecl param_strcpy(char *dst, const char *src)
{
  return stpcpy(dst, src) - (_DWORD)dst;
}
```

This is semantically equivalent but uses `stpcpy` (a more efficient variant that returns end pointer). At `-O0`, this is unexpected but not incorrect.
