# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are predominantly generic (`v0`, `v1`, `v2`, `v3`, `v4`) obscuring semantic purpose. For example, `local_array` uses `v1`, `v2`, `v3` instead of `index`, `value`, `ptr`. Some better names exist (`tree_node`, `head`, `status`) but are exceptions.

**2. Function Naming Quality:** Score: 8/10  
Justification: Most original function names preserved accurately (`double_value`, `local_vars`, `heap_basic`, `test_stack_memory`). A few generic names remain (`sub_10598`, `init_proc`) but core functionality names are intact.

**3. Literal Clarity:** Score: 5/10  
Justification: Mix of clear literals (42, 10, 20) and opaque hex values (`0x14u`, `0x28u`, `0x128CD`). String references are cryptic (`s`, `asc_128CD`, `byte_12A5A`). Original comments explaining expected values are lost.

**4. Noise Reduction:** Score: 6/10  
Justification: Register annotations (`// r0`, `// r3`) add visual noise. 18 library functions failed to decompile. Some optimization artifacts present (constant folding where original showed computation steps).

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: Aggressive constant folding obscures original logic (`local_vars` returns `2 * x + 10` directly, `stack_alias` returns constant `20`). `test_stack_memory` has constant-folded printf values instead of actual function calls, losing execution flow visibility.

**6. Function Decomposition:** Score: 7/10  
Justification: Function boundaries generally preserved with reasonable granularity. Complex functions like `linked_list_heap` maintain structure. Some functions over-optimized (`local_struct` collapsed to single return).

**7. Code Linearization:** Score: 7/10  
Justification: Most code reads top-to-bottom naturally. Loop structures recovered as do-while/while constructs. No excessive goto usage. Some forward jumps in complex functions but manageable.

**8. Redundancy Elimination:** Score: 7/10  
Justification: Minor redundancy present (`v2 = 0; v3 = 0;` in `local_array`). No severe loop unrolling. Some duplicate null checks but not obscuring logic significantly.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 6/10  
Justification: Basic types preserved (int, char, void*) but degradation visible (`Node*` becomes `_DWORD*` in `linked_list_heap`). Size types unclear (size_t vs int). Pointer types generally maintained.

**10. Composite Type Recovery:** Score: 7/10  
Justification: `Node` struct recovered with field names (`data`, `next`). `TreeNode` struct properly reconstructed. `Point` struct optimized away in some functions. Array types sometimes lost to pointers.

**11. Type Safety:** Score: 6/10  
Justification: Unsafe casts present (`(_DWORD *)`, `(int *)`). Pointer arithmetic visible in memory operations. No major type confusion but void* usage where specific types expected reduces safety.

## === LEVEL 4: SEMANTIC TRANSPARENCY ===

**12. Comment Quality & Presence:** Score: 3/10  
Justification: Nearly all original documentation lost (// MEM-L1-01, expected values, scene tags). Only minimal decompiler warnings present ("positive sp value has been detected"). No algorithm explanations recovered.

**13. Idiom Recognition:** Score: 5/10  
Justification: Memory allocation patterns recognizable (malloc/free pairs). Linked list and tree traversal patterns visible but not explicitly named or documented. No pattern identification provided.

**14. Anomaly Indication:** Score: 4/10  
Justification: 18 failed decompilations listed at end. Some stack warnings present. Critical security issues (dangling pointer, double-free) present without warnings. No indirect call resolution flags.

**15. Assertion & Contract Recovery:** Score: 3/10  
Justification: No precondition/postcondition documentation. Null checks present but not documented as contracts. Original expected return value comments completely lost.

## === LEVEL 5: CONTEXTUAL COHERENCE ===

**16. Cross-Reference Clarity:** Score: 6/10  
Justification: Function calls visible but not documented. Global variable usage visible (`global_counter`, `file_scope_static`). No call graph documentation. Callback relationships visible but unexplained.

**17. Module Organization:** Score: 7/10  
Justification: Functions appear in reasonable order matching source structure. Test functions grouped together. Static functions properly marked. No file organization metadata preserved.

**18. Build System Compatibility:** Score: 4/10  
Justification: 18 library functions failed to decompile requiring stubs. Platform-specific code (#ifdef) not preserved. Would require significant modification to compile standalone.
