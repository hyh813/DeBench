# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 2/10  
Justification: All variables use generic names (a1, a2, v1, v2, result) with zero semantic meaning recovered. Original names like `temp1`, `temp2`, `sum`, `count` are completely lost.

**2. Function Naming Quality:** Score: 6/10  
Justification: Core test functions preserved names (sequential_ops, recursion_factorial, double_value), but helper functions use generic labels (sub_1020, sub_1030, init_proc). Metadata mismatch (claims Ghidra, header says IDA Pro).

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers everywhere without explanation (0xFFFFFFFFLL, 4294967294LL, 100 * (a1 + 1)). No constants recovered from original source code.

**4. Noise Reduction:** Score: 5/10  
Justification: Compiler artifacts present throughout: stack canary checks (__readfsqword(0x28u)), excessive type casts ((unsigned int)), and register annotations (// rax, // r8d) clutter the code.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 6/10  
Justification: Simple functions recovered well (sequential_ops, single_if), but test functions show optimization artifacts—printing constants instead of actual function calls. duffs_device uses excessive goto labels.

**6. Function Decomposition:** Score: 7/10  
Justification: Functions properly separated with appropriate granularity. Helper functions (op_add, op_sub, state_idle) correctly identified as separate entities.

**7. Code Linearization:** Score: 5/10  
Justification: duffs_device function has 11 goto labels (LABEL_4 through LABEL_11) creating non-linear flow. Most other functions read top-to-bottom adequately.

**8. Redundancy Elimination:** Score: 6/10  
Justification: switch_fallthrough shows redundant pattern detection (hardcoded returns for cases 2 and 3 instead of fall-through logic). Some optimization artifacts remain visible.

## === LEVEL 3: TYPE SYSTEM FIDELITY (20% weight) ===

**9. Type Precision:** Score: 5/10  
Justification: __int64 returns where int was expected, mixed signed/unsigned throughout (int vs unsigned int), function pointers use _QWORD instead of proper signatures. Original int types degraded.

**10. Composite Type Recovery:** Score: 5/10  
Justification: No structs/unions in this code to evaluate. Arrays shown as raw memory (v5[4], v3[5], v14[4]) without type information. Matrix in loop_multi_exit lost 2D structure.

**11. Type Safety:** Score: 4/10  
Justification: Multiple unsafe casts ((unsigned int)(a1 + a2)), function pointers use generic _QWORD type, signed/unsigned mixing in comparisons ((int)a2 >= (int)a1).
