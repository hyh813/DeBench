## Decompilation Readability Assessment

**1. Variable Naming Quality:** Score: 3/10  
Justification: Nearly all variables use generic names like `v1`, `v2`, `v3` throughout (e.g., `int64_t v1; // 0xa18` in `const_param`), making it impossible to infer semantic purpose. Original source had meaningful names like `sum`, `len`, `current`.

**2. Function Naming Quality:** Score: 8/10  
Justification: Most function names were correctly recovered from debug symbols (`process_char`, `array_1d_stack`, `struct_simple`), matching original source. However, some helper functions are named generically (`function_15a0`, `function_1cbc`).
