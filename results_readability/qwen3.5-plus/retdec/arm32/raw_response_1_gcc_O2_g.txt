# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: All variables use generic names (`v1`, `v2`, `v3`...`v19` in duffs_device) with no semantic meaning. Original source had meaningful names like `temp1`, `matrix`, `sum`, `count` that indicated purpose.

**2. Function Naming Quality:** Score: 5/10  
Justification: Some function names preserved (`recursion_factorial`, `duffs_device`, `loop_multi_exit`), but 15+ functions completely missing from decompilation (sequential_ops, single_if, if_else, nested_if_2, etc.). Test functions exist but don't call actual implementations.

**3. Literal Clarity:** Score: 4/10  
Justification: String literals stored as hex-encoded UTF-8 in globals (`g1`, `g2`, `g3`). Duff's device switch cases use wrong values (-2, -3, -4, 3, 2, 1, -1 instead of 0-7). Magic numbers throughout without explanation.

**4. Noise Reduction:** Score: 5/10  
Justification: Excessive compiler artifacts: explicit casts (`(int32_t)&matrix`), raw pointer arithmetic (`v3 += 4`), stack canary calls (`__stack_chk_fail()`), unreachable labels (`lab_0x10d60`), and 19 redundant variables in duffs_device.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 4/10  
Justification: `recursion_factorial` converted from recursive to iterative (algorithm change). `computed_goto` returns address `0x11668` instead of implementing computed goto. Test functions print hardcoded values instead of calling functions. Duff's device switch structure is confusing with negative cases.

**6. Function Decomposition:** Score: 4/10  
Justification: Many functions missing entirely. Test functions bloated with hardcoded printf statements. `duffs_device` overly complex with 19 variables. Some functions transformed (recursive→iterative) breaking original structure.

**7. Code Linearization:** Score: 5/10  
Justification: Generally reads top-to-bottom but disrupted by forward jumps (`goto lab_0x10e50`), unreachable labels, and pointer arithmetic loops that obscure iteration patterns.

**8. Redundancy Elimination:** Score: 4/10  
Justification: Severe redundancy in `duffs_device` (v1-v19 largely duplicate). Test function loop variables duplicated unnecessarily. No dead code removal evident.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 6/10  
Justification: Consistent `int32_t` usage throughout. However, `jump_buffer` declared as `int32_t` instead of `jmp_buf`. Original had more semantic type distinctions (signed/unsigned based on context) that are lost.

**10. Composite Type Recovery:** Score: 5/10  
Justification: `matrix[3][4]` declared but initialization wrong (`{{1, 0, 0, 0}, {0}, {0}}` instead of actual values). Arrays declared but not properly initialized. No complex struct recovery issues since original lacked them.

**11. Type Safety:** Score: 4/10  
Justification: Multiple unsafe casts: `(int32_t)&matrix`, `(int32_t)&arr`, `(int32_t)&src`. `computed_goto` returns address as `int32_t`. Extensive pointer arithmetic without type safety checks.

## === LEVEL 4: SEMANTIC TRANSPARENCY ===

**12. Comment Quality & Presence:** Score: 2/10  
Justification: ZERO explanatory comments in decompiled code. Only meta-information (address ranges, line ranges). No algorithm explanations, confidence indicators, or logic documentation. Original had extensive comments.

**13. Idiom Recognition:** Score: 3/10  
Justification: Duff's device partially recovered but not identified as such. State machine patterns not recognized. No design pattern detection. Original explicitly documented control flow patterns.

**14. Anomaly Indication:** Score: 2/10  
Justification: No uncertainty flags anywhere. `computed_goto` confidently returns wrong value without warning. No indicators about unresolved calls, suspicious patterns, or decompilation confidence levels.

**15. Assertion & Contract Recovery:** Score: 2/10  
Justification: No preconditions, postconditions, or invariants documented. Error conditions unexplained. Original test validation comments completely lost.

## === LEVEL 5: CONTEXTUAL COHERENCE ===

**16. Cross-Reference Clarity:** Score: 4/10  
Justification: Function calls shown but many targets missing. Global variable usage unclear (`g1`, `g2`, `g3`, `g4`, `jump_buffer`). Callback relationships not documented. Test functions don't actually call tested functions.

**17. Module Organization:** Score: 4/10  
Justification: Functions grouped but not logically organized. Test functions separate but broken (hardcoded outputs). Missing functions break module coherence. No header file structure.

**18. Build System Compatibility:** Score: 5/10  
Justification: Includes headers and has prototypes. Uses compiler-specific features (`_setjmp` with struct initialization). External functions declared (`__longjmp_chk`, `__printf_chk`). Would require significant patching to compile standalone.
