## Decompilation Readability Assessment

### === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Nearly all local variables use generic names (`v1`, `v2`, `v3`, `result`) instead of semantic names like `sum`, `index`, or `pointer`. Function parameters sometimes preserve original names (`arr`, `n`, `p`) but locals are completely generic.

**2. Function Naming Quality:** Score: 7/10  
Justification: Most original function names preserved (`process_char`, `array_1d_stack`, `struct_simple`), but helper functions use mechanical names (`function_1080`, `function_1090`). Core test functions maintain semantic meaning.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers pervasive without explanation (`44`, `20`, `80` for struct offsets, `0x1c5d` for string addresses). Float constants loaded via pointer dereference instead of inline literals (`*(float32_t *)(v2 + 0x2341)`).

Justification: Significant compiler artifacts: `__x86_get_pc_thunk_*` calls, `__readgsdword(20)` stack canary checks, `__asm_rep_movsd_memcpy` intrusions. These obscure actual logic in nearly every function.

**5. Control Flow Naturalness:** Score: 5/10

**6. Function Decomposition:** Score: 7/10  
Justification: Function boundaries match original source well. Runtime helpers (`deregister_tm_clones`, `__do_global_dtors_aux`) are separate but don't interfere with user code understanding.

**7. Code Linearization:** Score: 6/10  
Justification: Generally reads top-to-bottom but complex functions like `array_3d` and `test_array_types` have nested while loops that disrupt flow. Forward jumps present in test functions.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Redundant variable assignments common (`v8 = result`, `v6 = v4` patterns). Same computation repeated in loop bodies instead of factored. No dead code elimination visible.

### === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 4/10  
Justification: Severe type erosion—everything becomes `int32_t` regardless of original type. `short`, `long`, `long long` all collapsed to `int32_t`. `const` and `volatile` qualifiers completely lost. Pointer types often just `int32_t`.

**10. Composite Type Recovery:** Score: 2/10  
Justification: **Critical failure**—NO struct/union types recovered. All struct access via raw pointer arithmetic (`*(int32_t *)(a1 + 4)`, `*(int32_t *)(a1 + 8)`). `Point3D`, `Rect`, `Widget`, `Node` completely lost. Field names absent.

**11. Type Safety:** Score: 3/10  
Justification: Extensive unsafe casts throughout (`(int32_t)arr`, `(char *)`). Pointer arithmetic without type safety. `ptr_func_simple` returns `(int32_t)f` instead of calling the function—semantic error.

**13. Idiom Recognition:** Score: 3/10

**14. Anomaly Indication:** Score: 4/10

**15. Assertion & Contract Recovery:** Score: 3/10
