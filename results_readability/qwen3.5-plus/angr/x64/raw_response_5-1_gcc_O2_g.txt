**1. Variable Naming Quality:** Score: 2/10  
Justification: Nearly all variables use generic names (`v0`, `v1`, `v2`, `ptr`, `p`) with no semantic meaning. Original variables like `capture_by_value`, `multiplier`, `instance_count`, `value`, `name` are completely lost. Only `index` and `result` appear occasionally.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 2/10  
Justification: Exception handling uses `goto` and raw `__cxa_*` calls instead of try-catch. `call_virtual_func` uses `goto *((long long *)*((long long *)&arg_0))` for vtable dispatch. Most test functions are empty stubs losing all original control flow.

**6. Function Decomposition:** Score: 3/10  
Justification: Severe over-fragmentation with single-line wrapper functions (`sub_40102d` just calls `sub_401030`). Many test functions are empty stubs despite having names. Thunk functions separated unnecessarily. Original cohesive test logic lost.

**7. Code Linearization:** Score: 4/10  
Justification: Exception handling has multiple forward jumps disrupting flow. Most functions too simple to assess. `test_cpp_oo_features` reads somewhat linearly with sequential printf calls but has unclear parameter passing (`v4`, `v5` unused).

**8. Redundancy Elimination:** Score: 3/10  
Justification: Significant redundancy from wrapper function chains. Thunk functions duplicated for each virtual method. Template instantiations create duplicate patterns without abstraction. Empty destructor stubs repeated for each class.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 2/10  
Justification: Severe type degradation: class types reduced to `void*` this pointers. Integer types mostly `unsigned long long` without proper signedness. Template functions have incorrect signatures (`template_swap<int>` takes single `unsigned int *ptr` instead of two references).

**10. Composite Type Recovery:** Score: 1/10  
Justification: Class definitions mostly empty (`class Base {} Base;`). Structs defined with generic fields (`field_0`, `field_8`). Original class fields (`value`, `name[32]`, `data`, `size`, `multiplier`, `shared_data`) completely lost. No inheritance relationships recovered.

**11. Type Safety:** Score: 2/10  
Justification: Numerous unsafe casts: `(long long)v3[8]`, `(char *)a0 + (*(a0))[24]`, `(unsigned long long)test_cpp_exception() & 4294967295`. Void* pointer arithmetic throughout. Vtable calls through raw pointer operations. No const-correctness preserved.
