# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 2/10  
Justification: All variables use generic names (v0, v1, a0, a1, a2) with zero semantic meaning. Original had descriptive names like `temp1`, `sum`, `count`, `i`, `j` that indicated purpose.

**2. Function Naming Quality:** Score: 6/10  
Justification: Main function names preserved (sequential_ops, single_if, nested_if_deep), but helper functions use mechanical names (sub_400894, sub_400970). Core functionality names match source.

**3. Literal Clarity:** Score: 2/10  
Justification: Magic numbers everywhere without explanation: 4294967295, 4294967294, 1717986919, 0xc0000000. Original arithmetic like `(a+b)*2-c` transformed to `(((a1+a0)*2|a1+a0>>31)&4294967294)-a2`.

**4. Noise Reduction:** Score: 3/10  
Justification: Significant compiler artifacts: `__ROL__`, `arm64g_calculate_condition`, `[D] unsupported_Iop_ZeroHI96ofV128()`. Redundant assignments like `v0=v2; v1=v3;` in _init function.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 3/10  
Justification: Simple loops transformed to bit operations (loop_for_fixed has no visible loop). Goto used inappropriately (`goto a0;` in call_func_ptr). Test functions print hardcoded values instead of calling actual functions.

**6. Function Decomposition:** Score: 5/10  
Justification: Function boundaries preserved but functionality incomplete. call_func_ptr_array only handles case 0, ignoring cases 1-2. Test functions don't actually test anything.

**7. Code Linearization:** Score: 4/10  
Justification: Multiple LABEL_ references disrupt flow (duffs_device has LABEL_4010c0 through LABEL_4010e0). Forward jumps in test_control_flow_l2 (LABEL_401460) break top-to-bottom reading.

**8. Redundancy Elimination:** Score: 4/10  
Justification: Duplicate function calls visible (call_weak_fn calls `__gmon_start__` twice, sub_400894 calls `_setjmp()` twice). Loop unrolling artifacts without abstraction.

## === LEVEL 3: TYPE SYSTEM FIDELITY (20% weight) ===

**9. Type Precision:** Score: 3/10  
Justification: Original `int` becomes `unsigned int` or `unsigned long long` inconsistently. Signedness lost (sequential_ops parameters should be signed int). Pointer types degraded to generic unsigned long.

**10. Composite Type Recovery:** Score: 2/10  
Justification: No struct recovery visible. Original `matrix[3][4]` in loop_multi_exit becomes pointer arithmetic with `g_4019cc` and `g_40199c`. Function pointer arrays not recovered (call_func_ptr_array incomplete).

**11. Type Safety:** Score: 3/10  
Justification: Unsafe `goto a0;` in call_func_ptr treats integer as code address. Multiple implicit casts throughout. No const correctness preserved. Function pointer signatures incorrect.

## === LEVEL 4: SEMANTIC TRANSPARENCY (15% weight) ===

**12. Comment Quality & Presence:** Score: 1/10  
Justification: Zero comments in decompiled output. Original had extensive documentation explaining each function's purpose, test cases, and expected behavior. No decompilation confidence indicators.

**13. Idiom Recognition:** Score: 2/10  
Justification: No pattern recognition. State machines appear as regular switch statements. Loop patterns transformed beyond recognition. No algorithm identification (factorial, digit counting, etc.).

**14. Anomaly Indication:** Score: 4/10  
Justification: Some anomalies marked (`[D] unsupported_Iop_ZeroHI96ofV128()`) but without explanation of impact. No warnings about potentially incorrect decompilation or semantic changes.

**15. Assertion & Contract Recovery:** Score: 2/10  
Justification: No preconditions documented (e.g., n>0 for factorial). No postconditions inferred. Original test validation comments completely lost.

## === LEVEL 5: CONTEXTUAL COHERENCE (10% weight) ===

**16. Cross-Reference Clarity:** Score: 3/10  
Justification: Global variables referenced (g_401a20, g_412ff8) without explanation. No call graph documentation. Callback relationships unclear (process_with_callback signature confusing).

**17. Module Organization:** Score: 3/10  
Justification: All functions in single flat output. Runtime functions (_init, _fini, deregister_tm_clones) mixed with application logic. No logical grouping or header structure.

**18. Build System Compatibility:** Score: 3/10  
Justification: Metadata claims "Compilation Result: Yes" but unsupported operations present. External dependencies undocumented. Function pointer gotos won't work in standard C.
