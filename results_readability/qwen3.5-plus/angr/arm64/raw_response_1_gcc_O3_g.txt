# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are generically named `v0`, `v1`, `a0`, `a1` with no semantic meaning. Original source used meaningful names like `temp1`, `sum`, `count`, `result` which are completely lost (e.g., `loop_for_fixed` uses `v0-v10` instead of `sum`, `i`).

**2. Function Naming Quality:** Score: 6/10  
Justification: Core functions retain original names (`sequential_ops`, `single_if`, `nested_if_2`), but many helper functions use generic `sub_400954` naming and some have compiler artifacts like `non_local_jump.constprop.0`.

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers pervasive without explanation: `4294967295` (UINT_MAX), `1717986919` (division constant), `0x40000000` appear throughout. Original return values like `-1` become `4294967295` obscuring intent.

**4. Noise Reduction:** Score: 4/10  
Justification: Significant compiler artifacts: runtime init functions (`_init`, `_start`, `__do_global_dtors_aux`), wrapper functions (`sub_400e3c` just calls `nested_if_2`), unsupported operations marked `[D]`, and stack canary checks throughout.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: Basic conditionals preserved (`if_else`, `if_elseif_chain`), but loops transformed into complex unrolled code (`loop_for_fixed`, `goto_backward`). Switch statements become jump tables (`CSWTCH.55[a0]`) or computed gotos despite -O0 compilation.

**6. Function Decomposition:** Score: 5/10  
Justification: Over-fragmented with ~80+ functions vs ~40 in source. Many `sub_*` wrapper functions add no value. Core logic functions reasonable size but mixed with runtime code.

**7. Code Linearization:** Score: 6/10  
Justification: Most functions read top-to-bottom, but jump tables introduce labels (`LABEL_4016f0`, `LABEL_4016dc`) and computed gotos (`goto (&v1)[a0]`) disrupt flow in `loop_multi_exit`, `call_func_ptr_array`.

**8. Redundancy Elimination:** Score: 4/10  
Justification: Severe loop unrolling in `loop_for_fixed` (8+ similar conditional blocks), `goto_backward` has 12+ nearly identical factorial branches, `duffs_device` shows extensive code duplication.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 5/10  
Justification: Inconsistent types: `unsigned int`, `unsigned long`, `unsigned long long` mixed without justification. Original `int` becomes various unsigned types. Exotic types like `uint128_t` and `char*` appear without clear purpose.

**10. Composite Type Recovery:** Score: 3/10  
Justification: No struct/union definitions recovered. Jump tables represented as raw arrays (`CSWTCH.55[4]`, `g_414028`). Function pointer arrays lose type information entirely.

**11. Type Safety:** Score: 4/10  
Justification: Unsafe casts throughout: `(unsigned int)v0`, `(char)arm64g_calculate_condition()`. Function pointers called via `goto v2` instead of proper invocation. No bounds checking on array accesses.
