# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 2/10  
Justification: All variables use generic names like `v0`, `v1`, `v2` with register hints (`r0`, `lr`) that provide no semantic meaning. Original source had meaningful names like `result`, `encrypted`, `decrypted`, `canary` that are completely lost.

**2. Function Naming Quality:** Score: 5/10  
Justification: Some original function names preserved (`param_fake_branch`, `call_opaque_predicate`), but signatures are wrong (e.g., `param_fake_branch()` takes no parameters when it should take `int x`). Helper functions use mechanical names (`sub_400498`, `sub_400574`).

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers everywhere without explanation: `4294967295` (should be `-1` or `UINT_MAX`), `90` (XOR key), `0x77359400`. Original had `0xDEADBEEF` with clear context. No constants recovered.

**4. Noise Reduction:** Score: 3/10  
Justification: Extensive compiler artifacts: register spilling variables (`v4 = v5; v3 = v6;`), `armg_calculate_condition()` calls obscuring logic, unnecessary goto labels (`LABEL_4005c4`, `LABEL_4005d4`).

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 3/10  
Justification: Excessive `goto` usage instead of structured control flow. Loops expressed as `do-while` with manual increments instead of `for` loops. Original `if (x * x < 0)` dead code completely eliminated rather than preserved.

**6. Function Decomposition:** Score: 5/10  
Justification: Function boundaries mostly preserved, but some are broken (`param_fake_branch` loses its parameter entirely). Constant folding eliminated logic (`call_fake_branch` just returns `10`).

**7. Code Linearization:** Score: 4/10  
Justification: Frequent forward jumps disrupt reading flow. Multiple exit points via goto labels. Original linear structure replaced with jump-heavy assembly-like patterns.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Some redundant assignments from register preservation (`v1 = v2; v0 = v3;`). However, some optimization artifacts like constant folding went too far, losing original logic structure.
