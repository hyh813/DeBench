## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: Basic control flow is recoverable (if/else, loops), but `func_mixed_args` has broken control flow with unsupported operations. Constant folding eliminates actual function calls, replacing dynamic logic with hardcoded returns.

**6. Function Decomposition:** Score: 4/10  
Justification: Poor decomposition with redundant wrapper functions (e.g., `sub_401219` just calls `call_cdecl`). Original 665 LOC expanded to 1787 LOC with many single-purpose wrapper functions that add no value.

**7. Code Linearization:** Score: 6/10  
Justification: Code generally reads top-to-bottom within functions, but the proliferation of wrapper functions disrupts overall flow. No excessive goto usage in recovered logic.

**8. Redundancy Elimination:** Score: 4/10  
Justification: Significant redundancy from constant folding (every `call_*` function returns hardcoded values instead of calling underlying functions). Wrapper function pairs (`sub_XXXXX` + actual function) duplicate structure unnecessarily.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 3/10  
Justification: Severe type degradation—`double` parameters become `unsigned int`, `long long` lost, pointer types reduced to `unsigned int*`. Original `func_mixed_args(int, char*, double, long long)` becomes `func_mixed_args(unsigned int, char*, unsigned int, unsigned int, unsigned int)`.

**10. Composite Type Recovery:** Score: 2/10  
Justification: Struct recovery is catastrophic. `LargeStruct` with `data[16]` becomes 32 separate `unsigned int` parameters. `SmallStruct` becomes `unsigned int*` with `a0[0] * a0[1]`. All struct field names lost (`field_0`, `field_4`, `padding_0`).

**11. Type Safety:** Score: 3/10  
Justification: Multiple unsafe patterns: function pointers treated as `unsigned int*`, void* casts implicit, pointer arithmetic on generic types. `ret_opaque_handle` returns `int` instead of `void*`, losing type safety entirely.
