**1. Variable Naming Quality:** Score: 2/10

**6. Function Decomposition:** Score: 4/10  
Justification: Test functions collapsed to constant-printing stubs losing actual logic. Runtime functions (`__muldf3`) are 200+ line monoliths that should be external library calls.

**7. Code Linearization:** Score: 5/10  
Justification: Simple functions read reasonably top-to-bottom, but floating point operations have excessive forward/backward jumps. Do-while patterns disrupt natural reading flow.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Floating point functions repeat identical bit-manipulation patterns. Test functions have redundant printf calls with pre-computed constants instead of actual function calls.

### === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 3/10

Justification: Complete struct/union failure - all accessed via pointer arithmetic (`param_1[1]`, `param_1[2]`, `param_1 + 0x14`). No field names recovered. Nested structures completely opaque.

**11. Type Safety:** Score: 3/10  
Justification: Pervasive unsafe casts: `(int *)`, `(undefined4 *)`, `(ulonglong)`. Pointer arithmetic on raw integers. Function pointers typed as `code *` losing signature information.
