## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 4/10

**6. Function Decomposition:** Score: 5/10  
Justification: Test functions maintain boundaries, but class methods fragmented (multiple `MiddleA::~MiddleA` variants). Helper functions (`FUN_00011xxx`) appear to be compiler thunks.

**7. Code Linearization:** Score: 5/10  
Justification: Stack checks at function exits disrupt flow. Exception handling creates non-linear jumps. Most functions read sequentially otherwise.

**8. Redundancy Elimination:** Score: 4/10  
Justification: Multiple destructor variants for same class with nearly identical code. `MiddleA::~MiddleA` and `MiddleB::~MiddleB` have duplicated logic patterns.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**11. Type Safety:** Score: 2/10
