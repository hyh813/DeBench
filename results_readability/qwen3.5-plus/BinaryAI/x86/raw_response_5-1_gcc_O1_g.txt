**7. Code Linearization:** Score: 6/10  
Justification: Most functions read top-to-bottom naturally. Exception handling and stack canary checks create disruptive early-exit patterns. No severe forward jumps except in exception unwinding code.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Some functions constant-folded to single returns (`test_cpp_multiple_inheritance` returns `0x47` directly). Multiple destructor thunks create redundancy. Runtime support functions duplicated across compilation units.
