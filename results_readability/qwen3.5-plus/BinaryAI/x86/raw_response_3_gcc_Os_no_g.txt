# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are overwhelmingly generic (`param_1`, `iVar1`, `uVar2`, `local_10`, `puVar5`) with no semantic meaning. Array names like `aiStack_38` indicate storage location but not purpose (should be `arr` or `buffer`).

**2. Function Naming Quality:** Score: 6/10  
Justification: Core test functions preserved original names (`double_value`, `local_vars`, `heap_basic`), but library calls degraded to `FUN_000111b0` (malloc) and `FUN_00011150` (free), obscuring their purpose.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers like `0x2a` (42), `0x14` (20), `0xf` (15) appear without explanation. String literals preserved well, but hex constants require manual conversion to understand.

**4. Noise Reduction:** Score: 4/10  
Justification: Stack canary checks add significant noise (`local_10 = *(int *)(in_GS_OFFSET + 0x14)` followed by validation). Generic function calls (`FUN_000111c0`) and redundant casts pollute the code.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 6/10  
Justification: Basic loops preserved as do-while constructs, but VLA/alloca functions degenerate into complex pointer arithmetic. Original `for` loops often transformed into less readable patterns.

**6. Function Decomposition:** Score: 5/10  
Justification: Function boundaries preserved, but some logic lost entirely (`stack_alias` returns constant `0x14` instead of implementing alias detection). Signatures don't always match original intent.

**7. Code Linearization:** Score: 7/10  
Justification: Code generally reads top-to-bottom with clear execution flow. Stack canary checks introduce branching but don't severely disrupt readability.

**8. Redundancy Elimination:** Score: 6/10  
Justification: Stack canary patterns repeat across functions without consolidation. Some expressions duplicated, but no severe dead code observed.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**10. Composite Type Recovery:** Score: 2/10

**11. Type Safety:** Score: 3/10
