# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are almost entirely generic (`param_1`, `param_2`, `iVar1`, `uVar3`) with no semantic meaning. Original source had meaningful names like `temp1`, `sum`, `count`, `result` that are completely lost. Complex functions like `loop_continue` have 9+ generic variables (uVar4-uVar9) obscuring simple odd-number summation logic.

**2. Function Naming Quality:** Score: 8/10  
Justification: Most function names are preserved from source (`sequential_ops`, `single_if`, `nested_if_2`, `loop_for_fixed`). Test functions also preserved (`test_control_flow_l1`). Only startup/runtime functions have generic names (`FUN_00101020`, `_init`).

**3. Literal Clarity:** Score: 5/10

**4. Noise Reduction:** Score: 6/10  
Justification: Vector operation artifacts create noise (`auVar13`, `auVar14` arrays in `recursion_factorial`). Some WARNING comments present ("Could not recover jumptable"). Simple functions are clean, but optimized functions have significant compiler artifact pollution.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 4/10  
Justification: Critical failure - compiled at `-O0` but decompiled code shows aggressive optimization artifacts. `loop_continue` uses bit manipulation instead of simple for-loop. `recursion_factorial` and `goto_backward` are completely unrolled with vector operations. `duffs_device` uses goto labels instead of structured switch.

**6. Function Decomposition:** Score: 7/10  
Justification: Function boundaries preserved correctly. Helper functions (`double_value`, `triple_value`) remain separate. No inappropriate inlining detected. Function sizes are reasonable and match source granularity.

**7. Code Linearization:** Score: 6/10  
Justification: Most functions read top-to-bottom naturally. `duffs_device` has disruptive goto labels (`switchD_00101717_caseD_7`). Test functions are very linear but hardcoded. Complex optimized functions have non-linear bit manipulation sequences.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Significant redundancy from loop unrolling in `recursion_factorial`, `tail_recursion`, `goto_backward`. Vector operations duplicate similar computation patterns 4+ times. Test functions eliminate all computation entirely (just printf with constants).
