# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are predominantly generic (`iVar1`, `iVar2`, `pbVar3`, `bVar2`) with no semantic meaning. Original names like `decrypted`, `buffer`, `valid` are preserved in some functions, but most local variables lost their intent.

**2. Function Naming Quality:** Score: 8/10  
Justification: Main function names preserved accurately (`param_fake_branch`, `param_opaque_predicate`, `decrypt_string`). Helper functions use generic names (`FUN_00011030`) but core logic functions retain semantic meaning.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers abound without explanation: `0x5a` (XOR key), `0x1f` (buffer size), `signal(8,...)` instead of `SIGFPE`. Return values constant-folded to hex (`0xe1`, `0x14`) obscuring original calculations.

**4. Noise Reduction:** Score: 5/10  
Justification: Compiler artifacts present: `__x86.get_pc_thunk.bx` warnings, `__stack_chk_fail_local()` calls, and "Unresolved local var" comments throughout. Three duplicate definitions of `param_division_by_zero` create confusion.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 6/10  
Justification: Loops recovered as do-while constructs appropriately. However, `param_fake_branch` lost its fake branch logic entirely (just returns `x`), and `call_fake_branch` constant-folded to `return 10`, losing original structure.

**6. Function Decomposition:** Score: 6/10  
Justification: Function granularity reasonable, but caller functions collapsed due to constant folding (`call_opaque_predicate` inlines the GCD loop). Multiple versions of same function (`param_division_by_zero` x3) indicate decomposition issues.

**7. Code Linearization:** Score: 7/10  
Justification: Code generally reads top-to-bottom with minimal forward jumps. Do-while loops instead of for-loops slightly disrupt flow but remain comprehensible.

**8. Redundancy Elimination:** Score: 4/10  
Justification: Significant redundancy: `test_obf_opt_edge` duplicates code from individual call functions. Three versions of `param_division_by_zero` with different implementations. No dead code removal for unreachable branches.
