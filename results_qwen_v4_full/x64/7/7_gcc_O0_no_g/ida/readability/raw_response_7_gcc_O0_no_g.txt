# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30%) ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are generic (`a1`, `v2`, `v3`, `v4`) instead of semantic names like `x`, `result`, `p1`, `p2`. At `-O0` with debug symbols, variable names should be recoverable but are lost.

**2. Function Naming Quality:** Score: 9/10  
Justification: Function names preserved from source (`param_fake_branch`, `param_opaque_predicate`, `decrypt_string`). Clear action-oriented naming that matches original intent.

**3. Literal Clarity:** Score: 5/10  
Justification: Signal numbers `8` and `11` should be `SIGFPE`/`SIGSEGV`. Character literals `65`, `66` instead of `'A'`, `'B'`. XOR key `90` instead of `0x5A`.

**4. Noise Reduction:** Score: 6/10  
Justification: Stack canary code (`__readfsqword(0x28u)`) present throughout. Type casts `(unsigned int)`, `(unsigned __int64)` add visual noise but don't obscure logic.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25%) ===

**5. Control Flow Naturalness:** Score: 7/10  
Justification: Loop structures preserved correctly (for/while). Fake branches optimized away (expected). Signal handling with setjmp/longjmp preserved but uses numeric values.

**6. Function Decomposition:** Score: 8/10  
Justification: Appropriate function granularity with single responsibility. Helper functions (`decrypt_string`) properly separated. `call_*` wrappers preserved.

**7. Code Linearization:** Score: 8/10  
Justification: Functions read top-to-bottom naturally. Standard control structures without excessive gotos. Stack variable declarations at function top (C style).

**8. Redundancy Elimination:** Score: 7/10  
Justification: Dead code from fake branches correctly eliminated. Some redundant type casts present. No obvious duplicate expressions obscuring logic.

## === LEVEL 3: TYPE SYSTEM FIDELITY (20%) ===

**9. Type Precision:** Score: 6/10

**10. Composite Type Recovery:** Score: 5/10  
Justification: Bitfield struct in `param_implementation_defined` optimized to constant `47` (structure lost). Arrays shown as `_DWORD v1[8]` - reasonable but no struct definitions visible.

**11. Type Safety:** Score: 5/10  
Justification: Multiple unsafe casts (`(__sighandler_t)`, `(__int64)`). Raw pointer arithmetic `*(_DWORD *)(4LL * i + a3)`. No buffer overflow warnings despite overflow code present.

**13. Idiom Recognition:** Score: 3/10

**14. Anomaly Indication:** Score: 4/10

**15. Assertion & Contract Recovery:** Score: 2/10

## === LEVEL 5: CONTEXTUAL COHERENCE (10%) ===
