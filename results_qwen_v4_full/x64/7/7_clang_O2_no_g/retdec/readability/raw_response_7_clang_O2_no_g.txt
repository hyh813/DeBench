# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: All variables use generic names (`a1`, `v1`, `v2`, `temp`) with no semantic meaning. Original `x`, `result`, `str`, `p1`, `p2` are completely lost, making it impossible to track variable purpose without tracing every usage.

**2. Function Naming Quality:** Score: 6/10  
Justification: Main function names preserved (`param_fake_branch`, `param_opaque_predicate`) but helper functions degraded to address-based names (`function_1030`, `function_1040`). Original `call_*` wrapper functions retained but serve no purpose.

**3. Literal Clarity:** Score: 4/10  
Justification: String literals stored as hex-encoded globals (`g1` = `"OBF-L4-02: %d..."`), XOR key `0x5A` appears as magic number `90` without explanation. Original `0xDEADBEEF` eliminated by optimizer without trace.

**4. Noise Reduction:** Score: 4/10  
Justification: Extensive `__asm_*` intrinsics (`__asm_movdqu`, `__asm_paddd`, `__asm_pxor`) obscure simple vectorized loop logic. Redundant type conversions (`(int32_t)a1`, `& 0xffffffff`) throughout all functions.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 4/10  
Justification: Excessive goto labels (`lab_0x1403`, `lab_0x159d`, `lab_0x1561`, `lab_0x1590`) replace natural loop constructs. Original simple `if` statements in `param_fake_branch` completely eliminated (optimizer removed dead code).

**6. Function Decomposition:** Score: 5/10  
Justification: Main functions preserved but `param_vectorized_loop` bloated to 300+ lines with SIMD unrolling. Helper functions like `call_fake_branch` are trivial constant returners that add no value.

**7. Code Linearization:** Score: 4/10  
Justification: Multiple forward jumps into loop bodies disrupt top-to-bottom reading. `param_vectorized_loop` has 6+ label targets scattered across 300 lines, making execution path nearly impossible to follow.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Loop unrolling in vectorized function not recognized or abstracted. Repeated SIMD operation sequences (`__asm_movdqu` + `__asm_paddd` + `__asm_movdqu_1`) appear multiple times without consolidation.

## === LEVEL 3: TYPE SYSTEM FIDELITY (20% weight) ===

**9. Type Precision:** Score: 4/10  
Justification: Original `int` universally widened to `int64_t`. Function signatures corrupted (`param_division_by_zero` has 3 parameters vs original 1). Signed/unsigned confusion throughout (`uint32_t v1 = (int32_t)a1`).

**10. Composite Type Recovery:** Score: 4/10  
Justification: `struct __jmp_buf_tag` recovered but with generic fields (`e0[8]`, `e1`, `e2`). Original bitfield struct in `param_implementation_defined` completely lost. Array vs pointer distinction frequently degraded.

**11. Type Safety:** Score: 4/10  
Justification: Pervasive unsafe casts (`(char *)str`, `(int32_t)a1`). Pointer arithmetic without bounds checking. No const-correctness preserved. Function pointer types degraded to generic integers.
