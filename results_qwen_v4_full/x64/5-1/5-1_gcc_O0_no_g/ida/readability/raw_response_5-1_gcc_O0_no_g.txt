# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 5/10  
Justification: Variables are mostly generic (`v0`, `v1`, `v2`, `a1`, `a2`) with occasional semantic names like `Value`, `Data`, `Size`. The `this` pointer is correctly identified for member functions, but local variables like `v3[40]` should be named `obj_buffer` based on usage as object storage.

**2. Function Naming Quality:** Score: 6/10  
Justification: Test functions recovered well (`test_cpp_member_func`, `test_cpp_constructor`), but class methods use mangled names (`_ZN11SimpleClassC2EiPKc`). Stub functions `sub_2020` through `sub_2170` have no semantic meaning.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers pervasive: `0x1234` becomes `4660` without explanation, `0x1Fu` for strncpy length, `40`/`80` for Container offsets. String literals like `"Test"`, `"Base"`, `"Derived"` preserved correctly.

**4. Noise Reduction:** Score: 4/10  
Justification: Stack canary checks (`__readfsqword(0x28u)`) in every function, 26 failed external function decompilations, 20+ stub functions (`sub_2020`-`sub_2170`), excessive type casts (`(_DWORD *)`, `(_QWORD *)`).

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 6/10  
Justification: Basic loops and conditionals preserved, but exception handling collapsed (`test_cpp_exception` only shows throw, no try-catch). Virtual dispatch shown as raw function pointer calls `(**(__int64 (***)())a1)(a1, a2)`.

**6. Function Decomposition:** Score: 6/10  
Justification: Test functions match source granularity, but STL template expansion creates excessive fragmentation (50+ unique_ptr helper functions). Constructor/destructor pairs properly separated.

**7. Code Linearization:** Score: 7/10  
Justification: Generally reads top-to-bottom with proper statement ordering. Stack variable declarations at function start disrupt flow slightly. No spurious gotos in main logic.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Massive STL template instantiation redundancy (multiple `__uniq_ptr_impl`, `_Tuple_impl`, `_Head_base` variants). Vtable setup code repeated across similar classes without abstraction.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 5/10  
Justification: Basic types recovered (`int`, `double`, `char*`), but generic types dominate (`_DWORD`, `_QWORD`, `_BYTE`). Const-correctness lost (`const char*` becomes `char*`). Function pointers degraded to `__int64 (***)()`.

**10. Composite Type Recovery:** Score: 5/10  
Justification: Class names recovered (`SimpleClass`, `LifecycleClass`, `Base`, `Derived`), but fields accessed via offset (`*(char *)this + 4`). Virtual inheritance shows complex pointer arithmetic (`*(_QWORD *)(*(_QWORD *)this - 24LL)`).

**11. Type Safety:** Score: 4/10  
Justification: Pervasive unsafe casts (`(_DWORD *)`, `(_QWORD *)`), raw pointer arithmetic throughout, function pointer calls without proper signatures. No const-correctness or bounds checking visible.

## === LEVEL 4: SEMANTIC TRANSPARENCY ===

**12. Comment Quality & Presence:** Score: 2/10  
Justification: Only function header comments with addresses. No algorithm explanations, no decompilation confidence indicators, no API contracts. Original source comments completely lost.

**13. Idiom Recognition:** Score: 3/10  
Justification: C++ idioms not identified: RAII pattern invisible, virtual dispatch shown as raw calls, template instantiation not recognized as such. Lambda captured as `::{lambda(int)#1}::operator()`.

**14. Anomaly Indication:** Score: 5/10  
Justification: Failed decompilations marked "FAILED to decompile" (26 functions). Warning present for `_start`: "positive sp value has been detected, the output may be wrong!" No uncertainty markers in recovered code.

**15. Assertion & Contract Recovery:** Score: 2/10  
Justification: No preconditions documented (e.g., `idx < size` checks not explained). No postconditions, no invariants. Exception handling contracts completely lost.

## === LEVEL 5: CONTEXTUAL COHERENCE ===

**16. Cross-Reference Clarity:** Score: 5/10  
Justification: Function calls visible, but virtual dispatch relationships unclear. Inheritance hierarchy not documented. Static member `instance_count` referenced without class context explanation.

**17. Module Organization:** Score: 5/10  
Justification: All functions in single output, no logical grouping. Related class methods scattered. No namespace or header file structure. Stub functions mixed with real code.

**18. Build System Compatibility:** Score: 3/10  
Justification: 26 external functions failed to decompile (printf, strlen, operator new, etc.). No dependency documentation. Cannot compile standalone without significant stubbing.
