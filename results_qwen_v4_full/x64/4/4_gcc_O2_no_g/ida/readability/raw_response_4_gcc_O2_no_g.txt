- **Composite Type Recovery (2/10)**

### Top 3 Recommendations for Decompiler Improvement

1. **Recover composite types from stack layouts**—Analyze parameter access patterns to reconstruct struct definitions instead of expanding to individual parameters (e.g., detect `a1[0]`, `a1[1]` access as struct fields)

2. **Preserve call relationships over constant folding**—Even when values are known at compile time, maintain function call structure to preserve semantic relationships between functions

3. **Recognize variadic function patterns**—Detect `va_start`/`va_arg`/`va_end` sequences or stack-based argument iteration and reconstruct proper `...` signatures with appropriate comments explaining the variadic handling
