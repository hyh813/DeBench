# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are predominantly generic (`a1`, `v0`, `v1`, `v2`, `ptr`) with no semantic meaning. Original source had descriptive names like `large_buf`, `local`, `arr`, but decompiled code uses `v2[2048]` and `v3[10]` instead.

**2. Function Naming Quality:** Score: 7/10  
Justification: Core test functions preserved original names (`local_vars`, `heap_basic`, `test_stack_memory`), but helper functions degraded to `sub_1030`, `sub_116C`. Function names generally indicate purpose when preserved.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers pervasive without explanation (`0x14u` for 20 bytes, `0x28u` for 40 bytes, `0xCu` for 12 bytes). Original used `sizeof(int)`, `sizeof(Point)` which were lost in decompilation.

**4. Noise Reduction:** Score: 6/10  
Justification: Compiler artifacts present: stack frame comments (`[esp+0h] [ebp-8h]`), redundant type casts (`(_DWORD *)`), register variables (`v0`, `v1`). Some cleanup achieved but not comprehensive.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 7/10  
Justification: Loop structures mostly recovered as `for` loops. Some functions use natural patterns (`for (i = 0; i < 10; ++i)`), but `stack_alias` lost its pointer comparison logic entirely.

**6. Function Decomposition:** Score: 7/10  
Justification: Function boundaries preserved well; each original function maps to one decompiled function. No inappropriate inlining or fragmentation observed.

**7. Code Linearization:** Score: 7/10  
Justification: Code reads top-to-bottom naturally in most cases. Early returns used appropriately. No excessive goto usage disrupting flow.

**8. Redundancy Elimination:** Score: 7/10  
Justification: Some redundant temporaries (`v2 = ptr[a1/2]; return v2;`), but no severe dead code. Compiler optimization artifacts manageable.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 4/10  
Justification: Severe type degradation: `int` becomes `_DWORD`, `Point*` becomes `_DWORD*`, `size_t` becomes `unsigned int`. Signed/unsigned distinctions often lost (`signed int nmemb`).

**14. Anomaly Indication:** Score: 5/10

**15. Assertion & Contract Recovery:** Score: 2/10

## === LEVEL 5: CONTEXTUAL COHERENCE ===

**16. Cross-Reference Clarity:** Score: 6/10

**18. Build System Compatibility:** Score: 4/10
