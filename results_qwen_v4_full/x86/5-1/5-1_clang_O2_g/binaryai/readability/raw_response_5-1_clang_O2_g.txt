# Decompilation Readability Assessment

## LEVEL 1: LEXICAL CLARITY (30% weight)

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are almost entirely generic (`local_28`, `uVar1`, `iVar1`, `puVar1`, `piVar2`) with no semantic meaning. Original source had meaningful names like `value`, `name`, `data`, `multiplier` that are completely lost.

**2. Function Naming Quality:** Score: 6/10  
Justification: Test function names preserved (`test_cpp_member_func`, `test_cpp_constructor`) but helper functions are generic (`FUN_00011030`, `FUN_00011385`). External functions marked `<EXTERNAL>::` lose original library names.

**3. Literal Clarity:** Score: 3/10  
Justification: Magic numbers everywhere without explanation: `0x125c`, `0x3e9`, `0x2a`, `0x47`, `0x28a`. String literals referenced as `DAT_000121d8` instead of actual content. Original had clear expected values documented.

**4. Noise Reduction:** Score: 5/10  
Justification: Compiler artifacts present: `__x86_get_pc_thunk_*`, `deregister_tm_clones`, `register_tm_clones`. Multiple stub functions that do nothing. Frame manipulation visible in `_start` function.

## LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight)

**5. Control Flow Naturalness:** Score: 5/10  
Justification: Basic control flow recovered but C++ semantics lost. Exception handling reduced to `__cxa_throw`/`__cxa_allocate_exception` calls. Virtual calls shown as `(***(code ***)param_1)` indirect calls without vtable explanation.

**6. Function Decomposition:** Score: 7/10  
Justification: Functions appropriately sized. Template instantiations recovered as separate functions (`template_max_int_`, `template_swap_int_`). Class methods use `__thiscall` convention correctly.

**7. Code Linearization:** Score: 6/10  
Justification: Code reads mostly top-to-bottom. Some forward jumps in `__do_global_dtors_aux` loop. Exception handling creates non-linear flow but generally traceable.

**8. Redundancy Elimination:** Score: 7/10  
Justification: `test_cpp_oo_features` repeats RTTI code from `test_cpp_rtti`. Otherwise clean without excessive duplication. No obvious dead code in main test functions.

## LEVEL 3: TYPE SYSTEM FIDELITY (20% weight)

**9. Type Precision:** Score: 4/10  
Justification: Heavy use of `undefined4`, `undefined`, `longdouble`. Pointer types often generic (`void*`, `int*`). Original `int`, `size_t`, `double` partially recovered but const-correctness completely lost.

**10. Composite Type Recovery:** Score: 3/10  
Justification: Class structures NOT recovered as structs. Object layout shown as pointer arithmetic (`this + 0x28`, `this + 0x50`). No field names (`value`, `name`, `data` all lost). Vtable structure not recovered.

**11. Type Safety:** Score: 3/10  
Justification: Many unsafe casts: `(undefined4 *)`, `(int *)`, `(char *)`. Pointer arithmetic without bounds checking. Type-confused operations common (treating objects as raw memory).
