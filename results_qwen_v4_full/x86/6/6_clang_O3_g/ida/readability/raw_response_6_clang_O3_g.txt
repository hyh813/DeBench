# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 5/10  
Justification: Parameters like `dst`, `src`, `filename` are preserved, but local variables use generic `v0`, `v1`, `v2`, `arg`, `ptr` patterns throughout (e.g., `param_malloc_free` has `v1` through `v9`), obscuring semantic purpose.

**2. Function Naming Quality:** Score: 7/10  
Justification: Core test functions retain original names (`param_strcpy`, `call_strcmp`, `thread_compute`), but initialization code uses mechanical names (`sub_1030`, `sub_13E0`, `_x86_get_pc_thunk_bx`).

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers pervasive without explanation: syscall numbers (5, 6), signal numbers (10, 14), socket constants (2, 1), and bitmasks (0x7F, 0xFF00) appear raw instead of named constants.

**4. Noise Reduction:** Score: 4/10  
Justification: 59 "FAILED to decompile" entries pollute output; compiler artifacts like `__x86.get_pc_thunk.bx`, `frame_dummy`, `deregister_tm_clones` included without filtering.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: Simple functions read naturally, but optimized functions show loop unrolling (`param_malloc_free`, `thread_atomic_increment`) and complex conditional chains that obscure original intent despite -O0 compilation claim.

**6. Function Decomposition:** Score: 6/10  
Justification: Original function boundaries mostly preserved, but `call_*` functions are constant-folded to return literals (e.g., `call_strcpy` returns `strlen(s)` directly as 8), losing call relationship semantics.

**7. Code Linearization:** Score: 6/10  
Justification: Most functions read top-to-bottom, but `param_thread_local_storage` has deeply nested conditionals with multiple early returns and goto-like label patterns (`LABEL_6`, `LABEL_13`).

**8. Redundancy Elimination:** Score: 5/10  
Justification: Loop unrolling in `thread_atomic_increment` creates 4x repetitive `_InterlockedAdd`/`_InterlockedCompareExchange` sequences; constant folding eliminates actual computation in test functions.
