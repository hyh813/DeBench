# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are generic (`param_1`, `param_2`, `iVar1`, `local_8`) with no semantic meaning. Original source had meaningful names like `count`, `args`, `sum`, `x`, `s` that are completely lost.

**2. Function Naming Quality:** Score: 8/10  
Justification: Most function names preserved from source (`cdecl_func`, `varargs_func`, `test_calling_conventions`). Some generic names exist (`FUN_00011030`, `FUN_00011080`) but core test functions are well-named.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers in hex (`0xf`, `0x32`, `0x2a`) instead of decimal. String literals replaced with `&DAT_00012352` references instead of actual strings like `"=== 测试基础调用约定 ==="`.

**4. Noise Reduction:** Score: 6/10  
Justification: Compiler artifacts present (`__i686.get_pc_thunk.bx`, `__do_global_dtors_aux`, `_init`, `_fini`). External function stubs are circular (printf calls printf).

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 7/10  
Justification: Loop structures mostly preserved (do-while patterns). However, constant folding obscures original logic (call_cdecl returns `0xf` directly instead of calling `cdecl_func(5, 10)`).

**6. Function Decomposition:** Score: 7/10  
Justification: Function boundaries match source. However, optimization merged some logic (caller functions return constants instead of making actual calls), losing call relationships.

**7. Code Linearization:** Score: 7/10  
Justification: Code reads top-to-bottom with minimal gotos. Stack variable access patterns (`stack0x00000008`, `aiStack_8c`) create minor disruption to flow.

**8. Redundancy Elimination:** Score: 6/10  
Justification: Constant folding created redundancy where computed values became literals. `test_calling_conventions` has 10+ printf calls with hardcoded values instead of function calls.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 6/10  
Justification: Basic types preserved but degradation exists. `func_mixed_args` splits `double` into `undefined4 param_4, param_5`. Return types sometimes changed (`int` to `undefined4`).

**10. Composite Type Recovery:** Score: 3/10  
Justification: Struct types completely lost (`LargeStruct`, `SmallStruct`, `BigStruct` not recovered). `func_struct_byval` uses raw stack access instead of struct field access. `func_struct_byptr` uses `int*` instead of `SmallStruct*`.

**11. Type Safety:** Score: 5/10  
Justification: Unsafe casts present (`CONCAT44`, `ROUND` in `func_mixed_args`). Pointer arithmetic without type safety. `param_double_ptr` has type confusion (`int*` vs `undefined4*`).
