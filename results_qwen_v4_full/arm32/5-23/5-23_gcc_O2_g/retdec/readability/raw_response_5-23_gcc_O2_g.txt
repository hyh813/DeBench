# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables use generic names like `v1`, `v2`, `v3`, `a`, `b` instead of semantic names (e.g., `page_size`, `exec_mem`, `input_array`). In `param_memory_protection`, `v2` should be `mem` or `page_ptr` based on mmap usage.

**2. Function Naming Quality:** Score: 5/10  
Justification: Some original names preserved (`test_preprocessing_features`, `param_memory_protection`), but many functions missing entirely and `__sync_fetch_and_add_4` is auto-generated. Only 6 of 20+ functions recovered.

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers throughout (`3`, `34`, `1`, `7` in mmap/mprotect calls) without symbolic constants like `PROT_READ`, `MAP_ANONYMOUS`. String literals stored as hex-encoded globals (`g1`, `g2`) instead of readable strings.

**4. Noise Reduction:** Score: 7/10  
Justification: Relatively clean without excessive compiler artifacts, though `__stack_chk_fail()` calls at function ends and unnecessary casts like `(int32_t *)&g1` add minor noise.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: `call_asm_simd` has broken loop structure with pointer arithmetic instead of array indexing. `__sync_fetch_and_add_4` uses unusual while loop that doesn't match original atomic operation semantics.

**6. Function Decomposition:** Score: 3/10  
Justification: Critical failure - only 6 functions recovered from 20+ original functions. Missing: `param_macro_constants`, `param_macro_functions`, `param_conditional_compile`, `param_asm_basic`, etc.

**7. Code Linearization:** Score: 6/10  
Justification: Code generally reads top-to-bottom, but `call_asm_simd` has awkward pointer chasing (`v2 += 4; v3 += 4`) instead of clean array iteration.

**8. Redundancy Elimination:** Score: 7/10  
Justification: No significant dead code observed, though this is partly because so much original code is missing entirely from decompilation.

## === LEVEL 3: TYPE SYSTEM FIDELITY (20% weight) ===

**9. Type Precision:** Score: 5/10  
Justification: Heavy use of `int32_t` throughout even where `size_t` (for sizes), `void*` (for generic pointers), or `char*` (for strings) would be more appropriate. Type erasure evident.

**10. Composite Type Recovery:** Score: 5/10  
Justification: Arrays recovered (`a[4]`, `b[4]`, `result[4]`) but no struct recovery. Original code had no complex structs, so limited opportunity here.

**11. Type Safety:** Score: 4/10  
Justification: Multiple unsafe casts (`(int32_t *)-1` for MAP_FAILED check, `(char *)&g1`). `call_asm_simd` has broken array initialization syntax (`a = {1, 0, 0, 0}` is invalid C).

## === LEVEL 4: SEMANTIC TRANSPARENCY (15% weight) ===

**12. Comment Quality & Presence:** Score: 1/10  
Justification: Zero comments in decompiled code. Original had extensive documentation explaining macro behavior, expected outputs, and assembly semantics. Complete loss of semantic context.

**13. Idiom Recognition:** Score: 2/10  
Justification: No pattern recognition evident. Atomic operations, SIMD patterns, memory protection idioms not identified or documented. `__sync_fetch_and_add_4` doesn't indicate atomic semantics.

**14. Anomaly Indication:** Score: 2/10  
Justification: No anomalies flagged despite obvious issues (broken array initialization, missing functions, `unknown_ffff0fc0` undefined function). Decompiler presents incorrect code confidently.

**15. Assertion & Contract Recovery:** Score: 2/10  
Justification: No preconditions, postconditions, or error handling documentation. Original had clear expected values (e.g., "期望: 64") completely lost.

## === LEVEL 5: CONTEXTUAL COHERENCE (10% weight) ===

**16. Cross-Reference Clarity:** Score: 3/10  
Justification: Function relationships unclear. Many called functions missing. Global variables (`g1`-`g20`) not explained - reader cannot know `g2` contains "PP-L2-01" format string without external analysis.

**17. Module Organization:** Score: 4/10  
Justification: Functions appear somewhat grouped (test functions together), but organization is poor. Missing function prototypes for many original functions. Header file structure lost.

**18. Build System Compatibility:** Score: 3/10  
Justification: Code will not compile as-is. Broken array initialization syntax (`a = {1, 0, 0, 0}`), undefined function `unknown_ffff0fc0`, missing includes for `printf`, `sysconf`, etc.
