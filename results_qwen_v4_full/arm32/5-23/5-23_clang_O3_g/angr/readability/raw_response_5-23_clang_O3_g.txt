# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are overwhelmingly generic (`v0`, `v1`, `a0`, `a1`, `ptr`) with no semantic meaning. Original source used meaningful names like `size`, `counter`, `increment`, `result` which are completely lost.

**2. Function Naming Quality:** Score: 7/10  
Justification: Core test functions preserve original names (`param_macro_constants`, `call_asm_basic`), but runtime helpers use cryptic names (`sub_4004ec`, `call_weak_fn`, `frame_dummy`).

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers like `0x400`, `0x200`, `57072` replace original constants (`MAX_SIZE`, `BUFFER_COUNT`). Some values are pre-computed (64, 30) losing calculation semantics.

**4. Noise Reduction:** Score: 4/10  
Justification: Runtime artifacts pollute output (`_init`, `deregister_tm_clones`, `register_tm_clones`, `__do_global_dtors_aux`). Variable assignments like `v0 = v1` followed by `goto` add unnecessary noise.

**7. Code Linearization:** Score: 6/10
