# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY (30% weight) ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are predominantly generic (`v1`, `v2`, `v3`, `result`, `mem`) with no semantic meaning. Original source had meaningful names like `large_buf`, `head`, `current`, `new_node` that are completely lost.

**2. Function Naming Quality:** Score: 6/10  
Justification: Core test functions preserved original names (`local_vars`, `heap_basic`, `test_stack_memory`), but many helpers are generic (`function_588`, `function_5ac`). `double_value` correctly preserved.

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers abound (2048, offsets like 0x13054) without explanation. String literals stored as hex-encoded globals (`g13`, `g15`) instead of readable strings. Error codes (-1, -2) are clear.

**4. Noise Reduction:** Score: 3/10  
Justification: Severe noise from compiler runtime (`__do_global_dtors_aux`, `register_tm_clones`, 40+ atomic sync functions). These weren't in original source and clutter the output significantly.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY (25% weight) ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: While loops recovered in some places, but `vla_stack` and `alloca_usage` have awkward non-idiomatic loop structures. Some functions constant-folded losing original control flow entirely.

**6. Function Decomposition:** Score: 5/10  
Justification: `test_stack_memory` inlines `large_stack_frame` code instead of calling it. Helper functions like `function_5ac` are just single-line wrappers around library calls (wasted decomposition).

**7. Code Linearization:** Score: 6/10  
Justification: Generally reads top-to-bottom with function prototypes at top. Some functions have complex branching that disrupts flow, but no excessive goto usage.

**8. Redundancy Elimination:** Score: 4/10  
Justification: 40+ atomic sync functions (`__sync_fetch_and_add_1/2/4`, etc.) are redundant variants. Compiler runtime code duplicated across multiple helper functions.

## === LEVEL 3: TYPE SYSTEM FIDELITY (20% weight) ===

**9. Type Precision:** Score: 4/10  
Justification: Uses `int32_t` consistently but loses original type distinctions. Pointer types often just `int32_t *` without proper struct types. Signed/unsigned sometimes incorrect (`uint32_t a1` in `global_array_access`).

**10. Composite Type Recovery:** Score: 2/10  
Justification: **Critical failure** - `Point`, `Node`, `TreeNode` structs completely lost. `heap_struct` returns `3 * a1` instead of accessing struct fields. `heap_nested` uses `*(int32_t *)(result + 4)` instead of `node->next`.

**11. Type Safety:** Score: 4/10  
Justification: Many unsafe casts present (`(int32_t *)a1`). Pointer arithmetic without type safety throughout. `memop_unaligned_access` does byte-level access that's type-unsafe.

## === LEVEL 4: SEMANTIC TRANSPARENCY (15% weight) ===

**12. Comment Quality & Presence:** Score: 1/10  
Justification: **Zero comments** in decompiled code. Original had extensive comments explaining each test case (MEM-L1-01, HEAP-L2-01, etc.). No decompilation confidence indicators.

**13. Idiom Recognition:** Score: 2/10  
Justification: Linked list creation in `linked_list_heap` not recognized as such. Tree traversal in `tree_heap_traversal` constant-folded to `return 60`. No pattern identification.

**14. Anomaly Indication:** Score: 3/10  
Justification: No anomalies flagged despite suspicious behavior (`stack_alias` just returns 20, losing alias logic). No indication of decompilation uncertainty or unresolved calls.

**15. Assertion & Contract Recovery:** Score: 3/10  
Justification: NULL checks present but not documented as contracts. No preconditions/postconditions. Original's safety checks (e.g., `if (n <= 0 || n > 1000) return -1`) preserved but unexplained.

## === LEVEL 5: CONTEXTUAL COHERENCE (10% weight) ===

**16. Cross-Reference Clarity:** Score: 4/10  
Justification: Function calls present but relationships not documented. Global variables (`g1` through `g23`) have no semantic meaning - original had `global_counter`, `global_array`, `file_scope_static`.

**17. Module Organization:** Score: 5/10  
Justification: All code in single file with prototypes at top. No logical grouping comments separating stack/heap/static sections that original had. Functions not grouped by category.

**18. Build System Compatibility:** Score: 5/10  
Justification: Has includes but many undefined symbols. `extern_function` defined at end but marked external in original. Atomic functions without proper headers. Metadata says "Compilation Result: Yes" but requires significant stubs.

**Overall Readability Score:** 4.1/10

### Critical Defects:
- **Struct types completely lost** - `Point`, `Node`, `TreeNode` reduced to pointer arithmetic (`*(int32_t *)(result + 4)`)
- **Zero comments** - Original had extensive documentation for each test case; decompiled has none
- **Global variables anonymized** - `global_counter`, `file_scope_static` became `g7`, `g4` with no semantic meaning
- **Compiler runtime pollution** - 40+ atomic sync functions and startup code clutter the output
- **Function inlining errors** - `test_stack_memory` inlines `large_stack_frame` instead of calling it

### Best Quality Aspect: **Function Naming Quality (6/10)**
- Core test functions preserved original names (`local_vars`, `heap_basic`, `double_value`), making basic navigation possible

### Most Needs Improvement: **Comment Quality & Presence (1/10)**
- Complete absence of comments makes understanding test purposes impossible without comparing to original source

### Top 3 Recommendations for Decompiler Improvement:
1. **Recover composite types from access patterns** - Detect struct field access patterns (`offset + 4`, `offset + 8`) and reconstruct `Point {x, y}`, `Node {data, next}`
2. **Preserve or infer global variable semantics** - Map `g7` → `global_counter`, `g4` → `file_scope_static` based on usage patterns
3. **Add decompilation confidence annotations** - Flag uncertain reconstructions (e.g., `// DECOMPILE UNCERTAIN: struct layout inferred`) and add section comments matching original organization
