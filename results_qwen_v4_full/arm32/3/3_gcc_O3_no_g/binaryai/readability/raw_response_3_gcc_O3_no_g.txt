## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: For-loops converted to do-while patterns. Some functions completely constant-folded (`stack_alias` returns `0x14` directly). `test_heap_memory` has goto label `LAB_000110fc` disrupting flow.

**6. Function Decomposition:** Score: 6/10  
Justification: Function boundaries mostly preserved, but many simplified beyond recognition (`local_array` reduced to `return param_1 * 5`). `tree_heap_traversal` logic significantly altered from source structure.

**7. Code Linearization:** Score: 6/10  
Justification: Generally reads top-to-bottom but complex functions like `test_heap_memory` have forward jumps and labels. Deep nesting in linked list functions with while(true) patterns.

**8. Redundancy Elimination:** Score: 6/10  
Justification: Pointer arithmetic repeated in loops (`(int *)((int)__ptr + -4)` pattern). Some constant folding created redundancy between actual computation and test expectations.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 4/10  
Justification: Heavy use of `undefined4`, generic `void*`, `int*` instead of specific types. Function pointers degraded to `code *` or `undefined4`. Lost `size_t`, `Point*`, `Node*`, `TreeNode*` type information.

**10. Composite Type Recovery:** Score: 3/10

**11. Type Safety:** Score: 4/10  
Justification: Many unsafe casts: `(int *)((int)__ptr + -4)`, `(undefined4 *)malloc(8)`. Pointer arithmetic without type safety. Function pointer calls through generic `code *` types.

## === LEVEL 4: SEMANTIC TRANSPARENCY ===

**12. Comment Quality & Presence:** Score: 1/10  
Justification: Zero comments in decompiled output. No algorithm explanations, no decompilation confidence indicators, no API contracts. Original source had extensive comments explaining each test case.

**13. Idiom Recognition:** Score: 3/10  
Justification: Common patterns (linked list traversal, tree traversal, memory allocation) not identified or named. No recognition of malloc/free pairs, no pattern documentation.

**14. Anomaly Indication:** Score: 3/10

**15. Assertion & Contract Recovery:** Score: 3/10  
Justification: NULL checks present but not documented as preconditions. No postcondition inference. Error conditions not documented (e.g., what `-1` return means).

## === LEVEL 5: CONTEXTUAL COHERENCE ===
