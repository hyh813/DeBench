# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are almost entirely generic (`param_1`, `param_2`, `iVar1`, `uVar2`, `piVar3`). No semantic meaning is recovered (e.g., `arr` becomes `piVar2`, `local` becomes `local_34`).

**2. Function Naming Quality:** Score: 7/10  
Justification: Most function names are preserved from source (`local_vars`, `heap_basic`, `global_var_access`). However, some signatures are corrupted (e.g., `local_array` returns `void` instead of `int`).

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers in hex obscure meaning (`0x2a` instead of `42`, `0x14` instead of `20`, `0xffffffff` instead of `-1`). Original decimal constants are lost.

**4. Noise Reduction:** Score: 5/10  
Justification: Stack canary checks (`__stack_chk_fail`) add noise. External function wrappers that call themselves (`calloc`, `memcpy`) are redundant artifacts.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 4/10  
Justification: For-loops converted to do-while with pointer arithmetic (`piVar2 = piVar2 + 1; *piVar2 = iVar2`). Test functions print hardcoded values instead of calling actual functions.

**6. Function Decomposition:** Score: 5/10  
Justification: Function boundaries preserved but signatures corrupted (`vla_stack` has extra `param_2`, `stack_alias` takes no parameters but should take two pointers).

**7. Code Linearization:** Score: 5/10  
Justification: Generally reads top-to-bottom but pointer arithmetic loops disrupt flow. Some functions have confusing early returns mixed with stack check failures.

**8. Redundancy Elimination:** Score: 6/10  
Justification: Some redundant pointer calculations (`(int *)((int)__ptr + -4)`). Stack canary variables (`aiStack_14[2]`) add unnecessary complexity.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 4/10  
Justification: `undefined4` used instead of `int`. Pointer types degraded to `void*` or `undefined4*`. Lost distinction between `size_t`, `int`, and pointer types.

**10. Composite Type Recovery:** Score: 2/10  
Justification: Struct `Point` completely lost (becomes `malloc(8)` with raw field access). `Node` and `TreeNode` structs not recovered—all pointer arithmetic.

**11. Type Safety:** Score: 3/10
