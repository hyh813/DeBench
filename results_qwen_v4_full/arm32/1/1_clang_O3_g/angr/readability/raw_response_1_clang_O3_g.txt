**6. Function Decomposition:** Score: 5/10

**7. Code Linearization:** Score: 3/10  
Justification: Excessive goto labels disrupt reading flow. Many functions have unreachable assignments after return statements. Control flow jumps around rather than following linear progression.

**8. Redundancy Elimination:** Score: 3/10  
Justification: Duplicate condition checks throughout. Unreachable code after returns. `armg_calculate_condition` calls repeat unnecessarily in multiple branches.
