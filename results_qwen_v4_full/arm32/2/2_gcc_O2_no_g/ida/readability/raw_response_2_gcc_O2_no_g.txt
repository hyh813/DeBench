# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: Variables are almost entirely generic (`v1`, `v2`, `a1`, `result`) with no semantic meaning. For example, `binary_tree_sum` uses `v1` through `v58` instead of meaningful names like `node`, `sum`, `left`, `right`. Original source had clear names like `sum`, `len`, `current`, `head`.

**2. Function Naming Quality:** Score: 6/10  
Justification: User functions preserved original names (`process_char`, `array_1d_stack`, `ptr_single`), but test functions lost their structure (constant-folded to print expected values). Internal helpers use cryptic names (`sub_103A0`, `sub_11718`).

**3. Literal Clarity:** Score: 5/10  
Justification: Magic numbers like `65`, `0x19`, `32` in `process_char` should be `'A'`, `'Z'`, and ASCII offset. Test functions show pre-computed constants (`97`, `300`, `10000LL`) instead of actual computation, obscuring the test logic.

**4. Noise Reduction:** Score: 4/10  
Justification: Significant compiler artifacts: register annotations (`r0`, `r3`, `lr`), stack frame comments (`[sp-4h] [bp-4h]`), type casts (`__fastcall`, `BYREF`), and runtime library functions (`__muldf3`, `__subdf3`) clutter the output.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 4/10  
Justification: `binary_tree_sum` transformed a simple 10-line recursive function into 200+ lines with 58 variables and nested loops. `enum_switch` uses array indexing instead of switch statement. Test functions lost all original control flow through constant folding.

**6. Function Decomposition:** Score: 5/10  
Justification: User functions have appropriate granularity, but `binary_tree_sum` is excessively complex (200+ lines). Test functions became monolithic constant-printing blocks. Runtime library functions incorrectly inlined.

**7. Code Linearization:** Score: 5/10  
Justification: Simple functions read top-to-bottom, but `binary_tree_sum` has extreme non-linearity with deeply nested structures. Multiple forward jumps disrupt flow in array processing functions.

**8. Redundancy Elimination:** Score: 4/10  
Justification: `binary_tree_sum` has massive redundancy with 58 variables for simple recursion. Test functions show redundant printf calls with pre-computed values. Loop patterns repeat across array functions without abstraction.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 6/10  
Justification: Basic types preserved (`int`, `float`, `double`), but struct types degraded to `_DWORD*` and `int*`. `volatile` qualifier lost in `volatile_access`. `long double` became `double`. Function pointer types partially recovered.

**10. Composite Type Recovery:** Score: 3/10  
Justification: Complete failure‚Äîstructs use array indexing (`a1[0]`, `a1[1]`, `a1[2]`) instead of field names (`p->x`, `p->y`, `p->z`). `struct_bitfields` uses manual bit manipulation. Unions and enums not recovered at all.

**11. Type Safety:** Score: 5/10  
Justification: Unsafe casts throughout (`(_DWORD *)`, `(int *)`, `(char *)`). Raw pointer arithmetic without bounds checking. Critical `volatile` qualifier loss in `volatile_access` changes semantics.
