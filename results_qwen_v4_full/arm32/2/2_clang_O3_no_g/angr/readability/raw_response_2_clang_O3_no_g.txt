## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 3/10  
Justification: Extensive goto usage throughout. Simple loops converted to do-while with goto patterns. Floating-point functions use `armg_calculate_condition` calls creating deeply nested conditional chains instead of natural if-else.

**6. Function Decomposition:** Score: 4/10  
Justification: Runtime functions excessively long (__muldf3 is 400+ lines). Test functions bloated with flag variables. Original simple functions (process_int: `return x * 2 + 1`) reasonably sized but logic sometimes obscured.

**7. Code Linearization:** Score: 3/10  
Justification: Frequent forward jumps disrupting flow. `LABEL_401xxx` patterns throughout. Functions like `__adddf3` have deeply nested gotos making top-to-bottom reading impossible.

**8. Redundancy Elimination:** Score: 4/10  
Justification: `armg_calculate_condition` calls repeat extensively in floating-point functions. Test functions have duplicate printf statements. Loop unrolling not recognized or commented.
