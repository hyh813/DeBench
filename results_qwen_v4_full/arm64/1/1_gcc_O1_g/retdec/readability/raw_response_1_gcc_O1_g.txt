# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 2/10  
Justification: All variables are generic (`v1`, `v2`, `v3`, `result`) with no semantic meaning. Original names like `temp1`, `sum`, `count` are completely lost. For example, `sequential_ops` uses `v1` instead of preserving `a`, `b`, `c` parameters.

**2. Function Naming Quality:** Score: 6/10  
Justification: Most original function names preserved (`loop_for_fixed`, `recursion_factorial`), but many helper functions are generic (`function_960`, `function_970`). Critical functions like `sequential_ops` exist but have broken signatures.

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers everywhere (`0x66666667` for division, `0xffffffff` masks). String literals stored as globals (`g7`, `g8`, `g9`) with hex-encoded Chinese characters instead of readable strings.

**4. Noise Reduction:** Score: 4/10  
Justification: Excessive type casting (`& 0xffffffff`), stack canary checks (`__stack_chk_fail()`), and global variable references (`g1`, `g12`) clutter every function. Compiler artifacts not eliminated.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 3/10  
Justification: Original `for`/`while` loops converted to `while` with complex conditions and `goto` labels (`lab_0x1348`, `lab_0x149c`). Switch statements often become if-else chains. Duff's device completely unrecognizable.

**6. Function Decomposition:** Score: 4/10  
Justification: Many functions have incorrect signatures - `sequential_ops(void)` should be `sequential_ops(int a, int b, int c)`. Parameter information lost in ~40% of functions, breaking semantic understanding.

**7. Code Linearization:** Score: 3/10  
Justification: Excessive forward jumps (`goto lab_0x149c`) disrupt top-to-bottom reading. Loop bodies scattered across multiple labels. Original linear flow completely obscured.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Repeated type masking operations throughout. Some dead code from optimization present but not severe. Loop unrolling not recognized or commented.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 4/10  
Justification: Original `int` becomes `int64_t` everywhere, even for 32-bit values. Signed/unsigned confusion (`uint32_t` vs `int32_t`). Function pointers degraded to generic types.

**10. Composite Type Recovery:** Score: 5/10  
Justification: `jmp_buf` structure partially recovered but with generic fields (`e0[8]`, `e1`, `e2`). No field names indicating purpose. Matrix arrays lost in `loop_multi_exit`.

**11. Type Safety:** Score: 4/10  
Justification: Multiple unsafe casts throughout. Pointer arithmetic obscured (`4 * v5 + v3`). Function pointer calls show incorrect signatures (`call_func_ptr` returns `x` instead of calling `f(x)`).

## === LEVEL 4: SEMANTIC TRANSPARENCY ===

**12. Comment Quality & Presence:** Score: 2/10  
Justification: Almost no comments except address ranges and line numbers. Original extensive documentation (test cases, difficulty ratings, scene labels) completely lost. No algorithm explanations.

**13. Idiom Recognition:** Score: 2/10  
Justification: Loop patterns not recognized as `for`/`while`. Switch fallthrough not indicated. Duff's device completely lost. State machine patterns not identified.

**14. Anomaly Indication:** Score: 2/10  
Justification: No warnings about uncertain decompilation. Incorrect code presented confidently (e.g., `sequential_ops` logic is completely wrong). No indication of potential issues.

**15. Assertion & Contract Recovery:** Score: 2/10  
Justification: No preconditions documented (e.g., `n > 0` for factorial). No postconditions inferred. Error conditions not documented. Original test validation comments lost.

## === LEVEL 5: CONTEXTUAL COHERENCE ===

**16. Cross-Reference Clarity:** Score: 4/10  
Justification: Function calls present but relationships unclear. Global variables (`g1`, `g12`) referenced without context. No call graph documentation. Callback relationships lost.

**17. Module Organization:** Score: 5/10  
Justification: All functions in single file. Some grouping by address range but no logical organization matching source structure. Test functions separated but not clearly.

**18. Build System Compatibility:** Score: 3/10  
Justification: Contains non-compilable assembly (`__asm_ld1`, `__asm_st1`). External dependencies not clearly documented. Some function signatures incompatible with original.

### Critical Defects:
- **Parameter loss**: `sequential_ops(void)` should be `sequential_ops(int a, int b, int c)` - completely breaks functionality

2. **Control flow reconstruction**: Better pattern matching for loop constructs (for/while/do-while) from jump patterns
3. **Semantic annotation**: Add confidence indicators and uncertainty warnings when decompilation is ambiguous or potentially incorrect
