# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 4/10  
Justification: Variables are predominantly generic (`v0`, `v1`, `ptr`, `i`) with no semantic meaning. Original source had meaningful names like `arr`, `local`, `p`, `head` that are completely lost.

**2. Function Naming Quality:** Score: 7/10  
Justification: Most user functions preserved original names (`local_vars`, `heap_basic`, `linked_list_heap`). However, runtime functions use generic names (`sub_400934`, `sub_400ab0`) and some are misnamed (`$x`).

**3. Literal Clarity:** Score: 3/10  
Justification: Pervasive magic numbers like `4294967295` (UINT_MAX), `4294967294`, `18446744073709551612` appear without explanation. Original source used clear constants (10, 2048, 100).

**4. Noise Reduction:** Score: 5/10  
Justification: Frame pointer manipulation visible throughout (`v4 = &v5` patterns). Redundant variable assignments and compiler artifacts from -O0 debug build pollute the code.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 6/10  
Justification: Loops generally recovered as for/while constructs. However, signed arithmetic uses awkward bit operations (`(a0 * 2 | a0 >> 31) & 4294967294`) instead of simple multiplication.

**6. Function Decomposition:** Score: 7/10  
Justification: Function granularity matches original source well. Runtime helper functions (`register_tm_clones`, `deregister_tm_clones`) are appropriately separated.

**7. Code Linearization:** Score: 6/10  
Justification: Most functions read top-to-bottom naturally. Some runtime functions use `goto` statements (`goto g_413fe0`) that disrupt flow.

**8. Redundancy Elimination:** Score: 5/10  
Justification: Signed overflow protection patterns repeat throughout (`| x >> 31) & 4294967294`). Some functions have duplicate return paths that could be consolidated.

## === LEVEL 3: TYPE SYSTEM FIDELITY ===

**9. Type Precision:** Score: 4/10  
Justification: Significant type degradation - `int` becomes `unsigned int`, pointers become `unsigned long long`. Original `Point` struct fields lost; everything becomes `field_0`, `field_8`.

**10. Composite Type Recovery:** Score: 4/10  
Justification: Structs recovered with generic field names (`field_0`, `field_8`, `padding_4`). Multiple conflicting `struct_0` definitions exist. Original `Node`, `TreeNode` structures partially lost.

**11. Type Safety:** Score: 4/10  
Justification: Unsafe casts throughout (`(char *)ptr + 4 * i`). Pointer arithmetic with void* and char* mixing. Some function signatures incorrect (`static_local` returns `unsigned int` vs original `int`).

## === LEVEL 4: SEMANTIC TRANSPARENCY ===

**12. Comment Quality & Presence:** Score: 1/10  
Justification: Zero comments in decompiled output. Original source had extensive documentation explaining each test case, expected values, and scenarios.

**13. Idiom Recognition:** Score: 2/10  
Justification: Common patterns (linked list traversal, tree creation, memory allocation) not identified or named. No algorithm recognition visible.

**14. Anomaly Indication:** Score: 2/10  
Justification: Dangerous code patterns (`dangling_pointer`, `double_free`, `heap_overflow`) not flagged. No warnings about undefined behavior or potential crashes.

**15. Assertion & Contract Recovery:** Score: 2/10  
Justification: NULL checks present but not documented as preconditions. No postcondition inference (e.g., "returns heap-allocated memory"). Error conditions undocumented.

## === LEVEL 5: CONTEXTUAL COHERENCE ===

**16. Cross-Reference Clarity:** Score: 5/10  
Justification: Function calls visible but relationships not documented. Global variables referenced (`$d.2`, `g_413ff8`) without explanation of purpose or origin.

**17. Module Organization:** Score: 6/10  
Justification: Functions appear in logical order matching source sections (stack, heap, static, memory ops). Test functions grouped appropriately.

**18. Build System Compatibility:** Score: 4/10  
Justification: External dependencies undeclared (`printf`, `malloc`, `fork`). Multiple undefined globals (`extern_global_var`, `extern_function`). Requires significant patching to compile standalone.
