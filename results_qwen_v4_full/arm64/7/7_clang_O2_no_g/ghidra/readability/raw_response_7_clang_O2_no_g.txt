# Decompilation Readability Assessment

## === LEVEL 1: LEXICAL CLARITY ===

**1. Variable Naming Quality:** Score: 3/10  
Justification: All variables use generic names like `iVar1`, `iVar2`, `param_1`, `uVar5` with no semantic meaning. For example, in `param_opaque_predicate`, the GCD variables should be named `gcd_result` or `a`, `b` based on the algorithm pattern.

**2. Function Naming Quality:** Score: 7/10  
Justification: Core test functions retain original names (`param_fake_branch`, `call_opaque_predicate`), but helper functions use generic names (`FUN_00100730`, `FUN_001008bc`). Runtime functions like `_init`, `_start` are standard but add noise.

**3. Literal Clarity:** Score: 4/10  
Justification: Magic numbers abound without explanation: `0x5a` (XOR key), `0x14` (should be 20), `0x23` (should be 35), signal numbers `8` and `0xb` instead of `SIGFPE`/`SIGSEGV`. The `0xDEADBEEF` constant from source is completely lost.

**4. Noise Reduction:** Score: 4/10  
Justification: Significant compiler artifacts: "WARNING: Removing unreachable block" appears 6+ times, runtime initialization code (`deregister_tm_clones`, `__cxa_finalize`) included, redundant casts like `(undefined *)0x0` and `(ulong)` conversions throughout.

## === LEVEL 2: STRUCTURAL INTELLIGIBILITY ===

**5. Control Flow Naturalness:** Score: 5/10  
Justification: Basic loops recovered (GCD do-while in `param_opaque_predicate`), but `param_vectorized_loop` has complex SIMD unrolling with labels (`LAB_00100c50`) that obscure the simple original `for` loop. Setjmp/longjmp pattern preserved but verbose.

**6. Function Decomposition:** Score: 5/10  
Justification: Critical failure: `param_fake_branch` reduced to empty `void` function (dead code eliminated), `call_fake_branch` constant-folded to `return 10`. `param_vectorized_loop` bloated to 150+ lines from original 15-line loop.

**7. Code Linearization:** Score: 6/10  
Justification: Most functions read top-to-bottom reasonably, but `param_vectorized_loop` has multiple forward jumps and loop labels that disrupt flow. Early returns used appropriately in error handlers.

**8. Redundancy Elimination:** Score: 4/10  
Justification: `call_opaque_predicate` duplicates the entire GCD calculation from `param_opaque_predicate` (inlining artifact). `param_vectorized_loop` has 8-element unrolled loops with 20+ accumulator variables (`iVar14` through `iVar24`).

**10. Composite Type Recovery:** Score: 3/10

**11. Type Safety:** Score: 4/10  
Justification: Multiple unsafe casts: `(undefined *)0x0`, `(ulong)local_40 | 1`, `(__sighandler_t)0x0`. Pointer arithmetic on `undefined8 *` types. No const-correctness preserved. Signal handler types cast without safety.
