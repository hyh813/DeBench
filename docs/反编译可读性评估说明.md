# Step1 可读性评估说明

Step1 评估反编译代码的可读性，由 LLM 作为 judge 完成。

## 1. 入口

主线调用：

- `scripts/run_single_task.py` / `scripts/auto_eval.py`
  - 通过 `scripts/pipeline_host.py` 在宿主机调用 `evaluator/readability/eval_readability.py`

## 2. 指标结构

当前使用五个 level：

- `L1_lexical_clarity`
- `L2_structural_intelligibility`
- `L3_type_system_fidelity`
- `L4_semantic_transparency`
- `L5_contextual_coherence`

并保留 18 个底层 dimensions 作为细分观测。

## 3. 分数口径

- `levels`：每个 level 内相关 dimensions 的平均值
- `overall_score`：五个 level 的**算术平均值**

说明：

- 已统一历史 Step1 结果的 `overall_score`
- 不再使用旧的加权平均口径

## 4. 主结果树中的 Step1 产物

每个 task 的 `readability/` 目录中保留：

- `raw_response_<eval_id>.txt`
- `test_results_*.json`

其中 `test_results_*.json` 是主线 pipeline 直接消费的结构化结果。

## 5. 状态同步

导入后，`eval_state_{arch}.json` 中的：

- `steps.readability.status`
- `steps.readability.score`

会与主结果树中的 Step1 产物保持一致。
